# 头脑风暴技能 (Brainstorming)

## 功能说明

此技能用于引导结构化的头脑风暴会议，帮助产生创意、解决问题和探索可能性。它能够提供：
- 系统化的创意思考框架
- 多角度的问题分析
- 创新解决方案的生成
- 想法的评估和优先级排序

## 使用场景

**在以下情况立即调用此技能：**
- 用户需要创意想法时
- 面临复杂问题需要解决方案时
- 需要产品功能创新时
- 用户提到"头脑风暴"、"创意"、"想法"、"brainstorming"等关键词
- 需要多角度思考问题时
- 团队需要集思广益时

## 核心原则

### 1. 头脑风暴四大基本原则
1. **追求数量**: 先求数量，再求质量
2. **禁止批评**: 不对任何想法进行评判
3. **欢迎疯狂**: 越疯狂的想法越好
4. **组合改进**: 在他人的想法基础上改进

### 2. 思维模式
- **发散思维**: 尽可能多地产生想法
- **收敛思维**: 评估和筛选最佳想法
- **横向思维**: 从不同角度思考问题

## 头脑风暴框架

### 阶段一：准备阶段（5-10 分钟）

#### 1.1 明确目标
```
我们要解决的问题是：[具体问题]
期望达到的目标是：[具体目标]
约束条件是：[时间、资源、技术等限制]
```

#### 1.2 设定范围
- 时间限制：通常 30-60 分钟
- 参与角色：定义相关利益方
- 输出形式：想法列表、思维导图等

### 阶段二：发散阶段（15-30 分钟）

#### 2.1 SCAMPER 法
对现有产品或想法进行以下思考：
- **S**ubstitute（替代）：什么可以被替代？
- **C**ombine（合并）：可以和什么合并？
- **A**dapt（改造）：可以如何改造？
- **M**odify（修改）：可以修改什么？
- **P**ut to other uses（他用）：还有什么其他用途？
- **E**liminate（消除）：可以去掉什么？
- **R**everse（反向）：如果反过来会怎样？

#### 2.2 六顶思考帽
从六个角度思考问题：
- ⚪ **白帽**：客观事实和数据
- 🔴 **红帽**：直觉和情感
- ⚫ **黑帽**：谨慎和风险评估
- 🟡 **黄帽**：乐观和积极面
- 🟢 **绿帽**：创新和可能性
- 🔵 **蓝帽**：控制和组织

#### 2.3 思维导图
```
中心主题
├── 分支 1
│   ├── 子分支 1.1
│   └── 子分支 1.2
├── 分支 2
└── 分支 3
```

#### 2.4 随机刺激法
- 随机选择一个词/图像
- 将其与问题关联
- 产生新的联想和想法

### 阶段三：收敛阶段（10-20 分钟）

#### 3.1 想法分类
将产生的想法按以下维度分类：
- **可行性**：高/中/低
- **影响力**：高/中/低
- **成本**：高/中/低
- **时间**：短期/中期/长期

#### 3.2 优先级矩阵
```
        高影响力
            │
    优先做   │  重点做
            │
────────────┼────────────
    考虑做   │  不做
            │
        低影响力
      低成本    高成本
```

#### 3.3 投票排序
- 每人 3-5 票
- 可以投给同一个想法
- 统计得票数
- 选出 Top 5-10 个想法

### 阶段四：深化阶段（10-15 分钟）

#### 4.1 想法完善
对选出的想法进行深化：
```
想法名称：[名称]
核心价值：[解决什么问题]
目标用户：[谁会使用]
实现方式：[如何实现]
所需资源：[需要什么]
预期效果：[能达到什么效果]
潜在风险：[可能的问题]
```

#### 4.2 行动计划
- 下一步行动
- 负责人
- 时间节点
- 成功指标

## 常用技巧

### 1. 5W2H 分析法
- **What**: 是什么？
- **Why**: 为什么？
- **Who**: 谁来做？
- **When**: 何时做？
- **Where**: 在哪里做？
- **How**: 怎么做？
- **How much**: 成本/效果如何？

### 2. 逆向思维
- 如果不这样做会怎样？
- 反过来的方案是什么？
- 最坏的情况是什么？

### 3. 类比思维
- 其他行业如何解决类似问题？
- 自然界有什么类似的解决方案？
- 历史上有过什么成功案例？

### 4. 极端场景法
- 如果用户量增加 100 倍？
- 如果预算减少 90%？
- 如果只有一天时间完成？

## 引导技巧

### 1. 打破僵局
- 从简单的热身问题开始
- 使用有趣的类比
- 分享一个相关的案例

### 2. 激发创意
- "如果没有任何限制，你会怎么做？"
- "如果是乔布斯/马斯克会怎么做？"
- "5 年后的解决方案会是什么样子？"

### 3. 处理沉默
- 使用轮流发言
- 提供思考时间（3-5 分钟静默思考）
- 使用书面头脑风暴（先写后讨论）

### 4. 管理冲突
- 强调对事不对人
- 寻找共同点
- 将冲突转化为建设性讨论

## 注意事项

1. **营造安全氛围**: 确保每个人都可以自由表达
2. **控制时间**: 严格遵守时间限制
3. **记录所有想法**: 不要遗漏任何点子
4. **鼓励多样性**: 欢迎不同背景和观点
5. **保持专注**: 避免偏离主题
6. **及时总结**: 每个阶段都要有小结
7. **跟进行动**: 确保想法能落地执行

## 示例场景

### 场景 1：产品功能创新
**用户**: "我们需要为产品增加一些创新功能"

**引导过程**:
1. 明确当前产品定位和用户群体
2. 使用 SCAMPER 法分析现有功能
3. 研究竞品的创新功能
4. 收集用户痛点和期望
5. 产生 20-30 个功能创意
6. 评估和筛选出 Top 5
7. 制定实施计划

### 场景 2：问题解决
**用户**: "我们的用户留存率很低，怎么办？"

**引导过程**:
1. 分析留存率低的具体数据
2. 使用 5Why 法找出根本原因
3. 从产品、运营、市场等角度分析
4. 产生可能的解决方案
5. 评估每个方案的成本和效果
6. 选择最优方案并制定行动计划

### 场景 3：战略规划
**用户**: "明年我们应该重点发展什么方向？"

**引导过程**:
1. 分析市场趋势和机会
2. 评估自身优势和劣势
3. 使用 SWOT 分析
4. 产生可能的战略方向
5. 评估每个方向的潜力和风险
6. 确定优先级和资源配置
