# PM 需求分析技能 (PRD Generator)

## 功能说明

此技能专门用于分析用户需求并生成完整的产品需求文档（PRD）。它能够帮助：
- 深入分析用户请求的核心需求
- 识别功能点和业务目标
- 生成结构化的 PRD 文档
- 提供技术实现建议

## 使用场景

**在以下情况立即调用此技能：**
- 用户提出新功能需求时
- 需要编写产品需求文档时
- 需要进行需求分析和拆解时
- 用户提到"PRD"、"需求文档"、"产品需求"等关键词
- 需要将模糊的需求转化为具体可实现的功能列表

## 使用方法

### 1. 需求收集
```markdown
请描述您的需求：
- 目标用户是谁？
- 要解决什么痛点？
- 期望实现什么功能？
- 有什么特殊要求或限制？
```

### 2. 需求分析框架

使用以下框架进行需求分析：

#### 2.1 用户故事
```
作为 [用户角色]
我想要 [功能]
以便于 [价值/好处]
```

#### 2.2 功能优先级（MoSCoW 法则）
- **Must have**: 必须有的功能
- **Should have**: 应该有的功能
- **Could have**: 可以有的功能
- **Won't have**: 本次不做的功能

#### 2.3 验收标准（INVEST 原则）
- Independent（独立的）
- Negotiable（可协商的）
- Valuable（有价值的）
- Estimable（可估算的）
- Small（小的）
- Testable（可测试的）

### 3. PRD 文档结构

生成包含以下内容的完整 PRD：

```markdown
# 产品需求文档

## 1. 文档信息
- 项目名称
- 版本号
- 创建日期
- 最后更新
- 负责人

## 2. 项目概述
### 2.1 项目背景
### 2.2 项目目标
### 2.3 目标用户
### 2.4 使用场景

## 3. 需求分析
### 3.1 用户故事
### 3.2 功能需求
#### 3.2.1 功能列表
#### 3.2.2 功能优先级
#### 3.2.3 功能详细描述
### 3.3 非功能需求
#### 3.3.1 性能要求
#### 3.3.2 安全要求
#### 3.3.3 兼容性要求

## 4. 产品设计
### 4.1 产品架构图
### 4.2 功能流程图
### 4.3 界面原型描述

## 5. 技术方案
### 5.1 技术选型
### 5.2 系统架构
### 5.3 数据模型
### 5.4 接口设计

## 6. 项目计划
### 6.1 里程碑
### 6.2 迭代计划
### 6.3 风险评估

## 7. 验收标准
### 7.1 功能验收
### 7.2 性能验收
### 7.3 质量指标

## 8. 附录
### 8.1 术语表
### 8.2 参考资料
```

## 输出要求

1. **结构化**: 使用清晰的标题层级和列表
2. **具体化**: 每个功能点都要有明确的描述和验收标准
3. **可执行**: 开发团队可以直接根据 PRD 进行开发
4. **完整性**: 覆盖所有关键方面，不遗漏重要信息
5. **专业性**: 使用标准的产品管理术语和框架

## 示例输出

当用户说"我需要一个用户登录功能"时，应该：

1. 调用此技能
2. 分析需求并生成完整的登录功能 PRD
3. 包括：
   - 用户故事
   - 功能列表（注册、登录、忘记密码、第三方登录等）
   - 安全要求（密码加密、验证码、防暴力破解等）
   - 验收标准
   - 技术实现建议

## 注意事项

- 主动询问用户不清楚的需求点
- 对于复杂需求，建议分阶段实现
- 始终考虑用户体验和技术可行性
- 提供多个方案供用户选择
- 标注潜在的技术风险和依赖
