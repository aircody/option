## 模块 2: 到期日选择器 (ExpirySelector)

### 功能描述
选择期权合约的到期日期，根据选中的股票代码动态获取该股票未来45天内的所有期权到期日，并标识特殊日期。

### 组件位置
`src/components/ExpirySelector.tsx`

### 功能规则

#### 2.1 动态获取到期日
- **触发时机**: 股票代码变更时
- **时间范围**: 未来45天内
- **数据来源**: 
  - Mock模式: 根据当前日期动态生成
  - 真实API: 调用LongPort API获取实际到期日

#### 2.2 特殊日期标识规则

| 标签 | 判定规则 | 特殊标记 | 优先级 |
|------|----------|----------|--------|
| 今日ODTE | 当日日期 | - | 最高 |
| 明日 | 下一个交易日 | - | 高 |
| 本周五 | 本周五（本周第5个交易日） | 绿色背景 | 中 |
| 下周五 | 下周五（下周第5个交易日） | 绿色背景 | 中 |
| 月度OPEX | 每月第3个周五（标准月度期权到期日） | 绿色背景 | 中 |
| 其他到期日 | 45天内的其他到期日 | 灰色背景 | 低 |

#### 2.3 日期计算逻辑
```
获取当前日期 (基准日期)
    ↓
计算未来45天内的所有日期
    ↓
筛选出期权到期日（通常是周五，但也可能有其他日期）
    ↓
标识特殊日期:
  - 今日: 当前日期
  - 明日: 当前日期 + 1天
  - 本周五: 本周的周五
  - 下周五: 下周的周五
  - 月度OPEX: 本月第3个周五
    ↓
按日期排序展示
```

#### 2.4 视觉规则
- **选中状态**: 蓝色背景 (#1890ff) + 白色文字
- **周五/OPEX**: 浅绿色背景 (#f6ffed) + 绿色边框 (#b7eb8f)
- **默认状态**: 灰色背景 (#f5f5f5) + 灰色边框 (#d9d9d9)
- **日期显示**: MM/DD 格式 (例如: 03/06)

#### 2.5 交互规则
- **点击**: 选中该到期日，触发数据刷新
- **默认选中**: 最近的一个到期日
- **禁用**: 加载数据时禁用选择

### 特殊日期判定算法

```typescript
// 判断是否为月度OPEX（第3个周五）
function isMonthlyOPEX(date: Date): boolean {
  const dayOfWeek = date.getDay(); // 0=周日, 5=周五
  if (dayOfWeek !== 5) return false;
  
  const weekOfMonth = Math.ceil(date.getDate() / 7);
  return weekOfMonth === 3;
}

// 判断是否为本周五
function isThisFriday(date: Date, today: Date): boolean {
  const dayOfWeek = date.getDay();
  if (dayOfWeek !== 5) return false;
  
  const diffDays = Math.floor((date.getTime() - today.getTime()) / (1000 * 60 * 60 * 24));
  return diffDays >= 0 && diffDays < 7;
}

// 判断是否为下周五
function isNextFriday(date: Date, today: Date): boolean {
  const dayOfWeek = date.getDay();
  if (dayOfWeek !== 5) return false;
  
  const diffDays = Math.floor((date.getTime() - today.getTime()) / (1000 * 60 * 60 * 24));
  return diffDays >= 7 && diffDays < 14;
}
```

## 模块 3: 关键指标卡片 (MetricCards)

### 功能描述
展示期权分析的核心指标数据。

### 组件位置
`src/components/MetricCards.tsx`

### 指标说明

#### 3.1 MAX PAIN (最大痛点)
- **含义**: 期权到期时，卖方整体利润最大、买方整体亏损最大的行权价，即卖方损失最小的价格点
- **显示**: `$606`
- **副信息**: 现价、涨跌幅
- **边框颜色**: 蓝色 (#1890ff)

**计算逻辑**:
```
对于每个候选行权价 K：
  Total Pain(K) = Σ Call Loss(K) + Σ Put Loss(K)
  
  其中：
  Call Loss(K) = max(S - K, 0) × 合约乘数(100) × Call OI(K)
  Put Loss(K) = max(K - S, 0) × 合约乘数(100) × Put OI(K)
  
  S = 假设的到期标的资产价格（即测试的每个候选行权价）

Max Pain = 使 Total Pain(K) 最小的那个 K
```

**示例**:
假设到期时标的价格 S = $220，行权价 K = $215 的 Call OI = 1000：
- Call Loss = max(220 - 215, 0) × 100 × 1000 = $500,000

假设行权价 K = $225 的 Put OI = 800：
- Put Loss = max(225 - 220, 0) × 100 × 800 = $400,000

#### 3.2 GAMMA EXPOSURE (Gamma敞口)
- **含义**: 做市商Gamma敞口，正值为多头，负值为空头
- **显示**: `-$42.13B`
- **状态标签**: 
  - 超波动 (红色): Gamma < -30
  - 正常 (绿色): Gamma >= -30
- **边框颜色**: 绿色 (#52c41a)

#### 3.3 PUT/CALL RATIO (看跌/看涨比率)
- **含义**: Put持仓量 / Call持仓量
- **显示**: `1.755`
- **解读**: 
  - > 1.5: 极度悲观 (反向强看多)
  - 0.7-1.5: 中性
  - < 0.7: 乐观
- **边框颜色**: 橙色 (#faad14)

#### 3.4 ATM IV (平值期权隐含波动率)
- **含义**: 平值期权的隐含波动率
- **显示**: `24.83%`
- **副信息**: HV (历史波动率)、VRP (波动率风险溢价)
- **边框颜色**: 紫色 (#722ed1)

#### 3.5 SKEW (25Δ RR) (波动率偏度)
- **含义**: 25 Delta Risk Reversal，反映下行保护成本
- **显示**: `+3.70%`
- **解读**:
  - > 3%: 适度偏斜，下行保护高于平均
  - < -3%: 偏斜向下
- **边框颜色**: 粉色 (#eb2f96)

## 模块 4: OI Wall 持仓墙图表

### 功能描述
展示各执行价的Call和Put持仓量分布，识别支撑阻力位。OI Wall（持仓墙）的核心是量化期权未平仓合约（OI）在特定行权价的集中程度，并基于期权对冲逻辑判定支撑/阻力。

### 组件位置
`src/components/OIWallChart.tsx`

### 核心概念

**阻力墙（Call OI Wall）**：Call 未平仓集中的行权价，做市商为对冲 Call 空头风险，会在价格逼近时卖出现货，压制上涨。

**支撑墙（Put OI Wall）**：Put 未平仓集中的行权价，做市商为对冲 Put 空头风险，会在价格逼近时买入现货，托举下跌。

### 识别方法（3种计算方式）

#### 方法1：相邻行权价对比法（最直观）
适用于快速识别图表中的"峰值"。

**公式**：
```
集中度比值(K) = OI(K) / ((OI(K-Δ) + OI(K+Δ)) / 2)
```
- Δ：行权价间隔（如 $5）
- OI(K-Δ)/OI(K+Δ)：K左右相邻行权价的 OI

**判定阈值**：比值 ≥ 2（即该行权价 OI 是相邻均值的 2 倍以上），即为 OI Wall。

**示例**：
$600 行权价 Put OI≈45k，相邻 $595/$605 Put OI 均值≈20k
比值 = 45k / 20k = 2.25 → 判定为强支撑墙

#### 方法2：相对占比法（适合量化全市场集中度）
适用于判断单一行权价 OI 在全市场的权重。

**公式**：
```
Call OI占比(K) = Call OI(K) / Σ所有行权价Call OI
Put OI占比(K) = Put OI(K) / Σ所有行权价Put OI
```

**判定阈值**：
- 高流动性标的：占比 ≥ 3%
- 低流动性标的：占比 ≥ 5%
- 且显著高于其他行权价，即为 OI Wall

#### 方法3：绝对阈值法（适合大盘股）
直接设定合约数阈值，简化计算。

**判定标准**：
- 美股大盘股（如 AAPL、TSLA）：Call/Put OI > 1万张
- 中盘股：OI > 5000张
- 且为峰值，即为 OI Wall

### 支撑/阻力强度计算

识别出 OI Wall 后，需计算强度（决定支撑/阻力的有效性）：

```
强度 = OI(K) × 合约乘数 × (1 / |S - K|)
```

**影响因子**：
- OI 数量越大，强度越高
- 行权价 K 与当前价格 S 越近，强度越高（近月期权尤为明显）
- 合约乘数固定（100），仅为标准化系数

### 图表展示规则

#### 4.1 图表类型
- **类型**: 双向柱状图
- **Call OI**: 红色柱 (#ff4d4f)，向上，表示阻力
- **Put OI**: 绿色柱 (#52c41a)，向下，表示支撑

#### 4.2 数据展示
- **X轴**: 执行价 (Strike)，单位：美元
- **Y轴**: 持仓量 (OI)，单位：k (千)
- **现价线**: 蓝色垂直虚线标识当前价格位置

#### 4.3 交互与解读
- **Tooltip**: 显示具体执行价的Call/Put持仓量
- **支撑识别**: Put OI 高点 = 支撑位（绿色峰值）
- **阻力识别**: Call OI 高点 = 阻力位（红色峰值）

**实战解读示例**：
- **支撑墙（绿色 Put OI）**：集中区间 $555-$600，核心峰值在 $600（OI≈45k）
  - 逻辑：价格跌至 $600 附近时，做市商买入现货对冲 Put 空头，形成强托底
  
- **阻力墙（红色 Call OI）**：集中区间 $615-$630，核心峰值在 $615（OI≈40k）
  - 逻辑：价格涨至 $615 附近时，做市商卖出现货对冲 Call 空头，形成强压制

### 关键注意事项

1. **到期日效应**：越接近期权到期日（尤其是当月期权），OI Wall 的支撑/阻力越强，做市商对冲行为越集中。

2. **动态性**：OI 随交易实时变化，若某行权价 OI 被大量平仓，持仓墙会瞬间失效，需实时更新数据。

3. **共振效应**：若 OI Wall 与 Max Pain（最大痛点）重合，该位置的支撑/阻力会呈指数级增强。

## 模块 5: Max Pain 曲线图表

### 功能描述
展示各执行价的卖方总损失（Total Pain）曲线，标识最大痛点（Max Pain）。

### 组件位置
`src/components/MaxPainChart.tsx`

### 核心概念

**Max Pain（最大痛点）**：期权到期时，标的资产价格使得期权卖方整体利润最大、买方整体亏损最大的行权价，即卖方损失最小的价格点。

**最大痛点曲线**：将每个候选行权价对应的"卖方总损失（Total Pain）"绘制成的曲线，曲线的最低点即为 Max Pain。

### 计算逻辑

#### 5.1 基础数据准备
- 到期日所有看涨期权（Call）和看跌期权（Put）的：
  - 行权价（K）
  - 未平仓合约数（OI）
  - 合约乘数（美股个股期权通常为 100 股/合约）

#### 5.2 计算公式

对于每个候选行权价 K，假设到期时标的资产价格为 S = K：

**看涨期权（Call）损失**：
```
Call Loss(K) = max(S - K, 0) × 合约乘数 × Call OI(K)
```
- 若 S ≤ K，期权不会被行权，卖方损失为 0

**看跌期权（Put）损失**：
```
Put Loss(K) = max(K - S, 0) × 合约乘数 × Put OI(K)
```
- 若 S ≥ K，期权不会被行权，卖方损失为 0

**总损失（Total Pain）**：
```
Total Pain(K) = Σ Call Loss(K) + Σ Put Loss(K)
```
（对所有行权价的 Call 和 Put 损失求和）

**确定 Max Pain**：
```
Max Pain = argmin(Total Pain(K)) for all K
```

#### 5.3 计算示例

假设有以下期权数据：
| 行权价 | Call OI | Put OI |
|--------|---------|--------|
| $215   | 1000    | 500    |
| $220   | 800     | 800    |
| $225   | 500     | 1200   |

计算 S = $220 时的 Total Pain：
- Call Loss @ $215: (220-215) × 100 × 1000 = $500,000
- Call Loss @ $220: max(220-220,0) × 100 × 800 = $0
- Call Loss @ $225: max(220-225,0) × 100 × 500 = $0
- Put Loss @ $215: max(215-220,0) × 100 × 500 = $0
- Put Loss @ $220: max(220-220,0) × 100 × 800 = $0
- Put Loss @ $225: (225-220) × 100 × 1200 = $600,000

Total Pain($220) = $500,000 + $0 + $0 + $0 + $0 + $600,000 = $1,100,000

遍历所有行权价后，Total Pain 最小的那个即为 Max Pain。

### 图表展示规则

#### 5.4 图表类型
- **类型**: 面积图 + 散点图
- **Total Pain**: 紫色面积 (#722ed1)，平滑曲线
- **Max Pain点**: 黄色圆点 (#faad14)，突出显示

#### 5.5 数据展示
- **X轴**: 执行价 (Strike)，单位：美元
- **Y轴**: Total Pain (卖方总损失)，单位：G (十亿美元)
- **曲线**: 连接每个行权价的 Total Pain 值形成的连续曲线

#### 5.6 交互功能
- **Tooltip**: 显示当前执行价的 Total Pain 值
- **Max Pain 标识**: 黄色圆点标记曲线的最低点
- **解读**: 
  - 曲线最低点 = Max Pain（卖方损失最小）
  - 价格通常向 Max Pain 靠拢（市场效应）

### 关键注意事项

1. **动态性**：Max Pain 基于到期日未平仓合约计算，随着期权交易、展期，未平仓数会变化，因此 Max Pain 也会动态调整，越接近到期日越稳定。

2. **市场影响**：部分投资者认为，标的价格会向 Max Pain 靠拢，因为做市商有动机通过对冲操作引导价格向该点移动，以最小化自身损失（但这并非必然，市场仍受多重因素影响）。

3. **数据依赖**：计算 Max Pain 需要准确的未平仓合约数据，通常来自交易所或专业金融终端。

## 模块 6: Gamma Exposure (GEX) 图表

### 功能描述
展示各执行价的Gamma敞口分布，计算全市场Gamma Exposure，识别Zero Gamma Level，并提供交易启示。

### 组件位置
- 图表组件: `src/components/GEXChart.tsx`
- 计算工具: `src/utils/gexCalculator.ts`

### 核心概念

**Gamma Exposure (GEX)**: 做市商整体持有的Gamma敞口，反映期权卖方（做市商）为维持Delta中性所需进行的对冲操作规模。

**Zero Gamma Level**: GEX为零的行权价位置，是正负Gamma的临界点。

**Gamma Flip Point**: 当价格穿越此点时，做市商的对冲行为从"抑制波动"转为"放大波动"（或反之）。

### 计算公式

#### 6.1 基础公式
```
GEX = Σ(Gamma × OI × 合约乘数 × 标的资产价格)
```

其中：
- **Gamma**: 期权的Gamma值（Delta的变化率）
- **OI**: 未平仓合约数
- **合约乘数**: 美股期权通常为100
- **标的资产价格**: 当前股票价格

#### 6.2 单个期权GEX计算
```
Call GEX = Gamma × Call OI × 100 × 标的价格
Put GEX = -Gamma × Put OI × 100 × 标的价格  (负号因为Put Delta为负)
```

#### 6.3 全市场GEX
```
Total GEX = Σ所有行权价的(Call GEX + Put GEX)
```

### Gamma估算模型

由于实际Gamma值需要复杂的期权定价模型，本系统使用简化模型估算：

| 距离ATM位置 | Gamma估算值 | 说明 |
|------------|------------|------|
| ATM (±1%) | 0.05 | ATM期权Gamma最高 |
| 近ATM (±3%) | 0.03 | 接近平值 |
| 中等OTM (±6%) | 0.015 | 中等距离 |
| 远OTM (>6%) | 0.005 | 深度虚值 |

### GEX状态判定

| 状态 | GEX范围 | 标签 | 颜色 | 市场影响 |
|------|---------|------|------|----------|
| 极端负Gamma | < -30B | 超波动 | 红色(#ff4d4f) | 做市商成为波动放大器，追涨杀跌 |
| 负Gamma | -30B ~ -10B | 高波动 | 浅红(#ff7875) | 波动可能放大，谨慎操作 |
| Gamma中性 | -10B ~ +10B | 平稳 | 绿色(#52c41a) | 市场相对稳定 |
| 正Gamma | +10B ~ +30B | 低波动 | 浅绿(#73d13d) | 做市商抑制波动，低买高卖 |
| 极端正Gamma | > +30B | 极稳 | 深绿(#389e0d) | 强烈抑制波动，市场盘整 |

### 负Gamma市场作用机制

Gamma是期权Delta的变化率，负Gamma意味着做市商（期权卖方）为维持Delta中性，需进行**同向对冲**：

**价格上涨时**：
- 期权Delta绝对值上升
- 做市商被迫买入标的
- 进一步推高价格（**助涨**）

**价格下跌时**：
- 期权Delta绝对值下降
- 做市商被迫卖空标的
- 进一步加剧下跌（**助跌**）

**极端效应**：
当Gamma绝对值达数十亿级别（如-$42.13B），叠加流动性萎缩，易触发"Gamma挤压"或"流动性崩盘"，形成趋势的自我强化。

### 交易启示

#### 6.4 策略适配建议

| 策略类型 | 适用性 | 核心操作 |
|----------|--------|----------|
| 趋势跟踪策略 | 高 | 顺应市场方向，下跌做空、上涨做多，避免逆势 |
| 波动率交易 | 高 | 买入波动率（跨式/宽跨式期权），或持有VIX看涨期权 |
| 均值回归策略 | 极低 | 负Gamma环境下失效，易被趋势碾压 |
| 期权卖方策略 | 禁止 | 波动放大带来指数级亏损，需平仓或对冲 |

#### 6.5 风险控制要点

1. **降低杠杆**: 负Gamma环境下价格滑点扩大，建议杠杆降至平时的50%以下
2. **加宽止损**: 止损幅度扩大至近期波动幅度的1.5-2倍
3. **关注到期日**: 临近到期（尤其是0DTE），Gamma非线性飙升，波动放大效应最强
4. **结合OI Wall与Max Pain**: 若负Gamma与关键位置重合，突破/跌破会引发极致波动

#### 6.6 负Gamma缓解条件

1. **期权到期**: 当月期权到期后，做市商负Gamma头寸自然清零
2. **头寸反转**: 大量投资者买入期权，做市商转为正Gamma
3. **流动性注入**: 大型机构主动吸收抛压/买盘，打破正反馈循环

## 模块 7: Put/Call Ratio (PCR) 图表

### 功能描述
展示各执行价的Put/Call Ratio分布，计算全市场PCR指标，识别市场情绪状态，并提供反向交易信号。

### 组件位置
- 图表组件: `src/components/PCRChart.tsx`
- 计算工具: `src/utils/pcrCalculator.ts`

### 核心概念

**Put/Call Ratio (PCR)**: 看跌期权与看涨期权的比值，是期权市场的经典情绪指标，通过统计Put与Call的成交量/未平仓合约量比值，量化市场多空情绪倾向。

**反向投资逻辑**: 期权市场中，散户/机构的极端看跌行为往往是情绪超调，并非真实基本面恶化。当看跌情绪达到极致，市场通常处于底部区域，此时反向操作（看多）的胜率极高。

### 计算公式

#### 7.1 基础公式
```
PCR = Put 成交量(OI) / Call 成交量(OI)
```

其中：
- **分子 (Put)**: 看跌期权的成交量或未平仓合约量 → 代表看空情绪
- **分母 (Call)**: 看涨期权的成交量或未平仓合约量 → 代表看多情绪

#### 7.2 数值含义
- **数值越高**: 市场看跌情绪越强烈
- **数值越低**: 市场看多情绪越占主导

### PCR 区间与信号对应

| PCR 数值区间 | 市场情绪 | 信号颜色 | 反向操作建议 |
|-------------|----------|----------|--------------|
| < 0.8 | 极度乐观 | 红色(#ff4d4f) | 反向看空（警惕高位回调） |
| 0.8 - 1.0 | 乐观 | 浅红(#ff7875) | 谨慎看空 |
| 1.0 - 1.2 | 中性 | 黄色(#faad14) | 观望/均衡策略 |
| 1.2 - 1.5 | 偏悲观 | 浅绿(#73d13d) | 谨慎看多 |
| > 1.5 | 极度悲观 | 绿色(#52c41a) | 反向强看多（核心信号） |

### 核心数值解读（以1.755为例）

#### 7.3 状态判定
- **核心数值**: 1.755
- **情绪状态**: 极度悲观 (反向强看多)
- **区间定位**: 处于 >1.5 的极端看跌区间，是典型的"情绪见底"信号

#### 7.4 反向投资逻辑
1. 期权市场中，散户/机构的极端看跌行为往往是情绪超调，并非真实基本面恶化
2. 当看跌情绪达到极致，市场通常处于底部区域
3. 此时反向操作（看多）的胜率极高

#### 7.5 OI数据解读
- **OI (Open Interest)**: 未平仓合约量，代表市场中未被平仓/行权的期权合约总数
- **高OI数值**（如2.106M）说明该比率对应的期权交易活跃度极高
- 情绪沉淀充分，信号可靠性远高于零星交易的短期情绪

### 结合市场环境的实操指引

#### 7.6 反向看多核心策略

**指数层面**:
- 配置标普500 (SPY)、纳斯达克 (QQQ) 等宽基指数的看涨期权
- 或做多指数ETF

**个股层面**:
- 回避基本面无问题的超跌标的
- 禁止盲目做空，避免在底部踏空

#### 7.7 风险控制（结合负Gamma环境）

1. **仓位控制**: 负Gamma环境下市场波动被剧烈放大，建议仓位控制在常规的50%以下
2. **严格止损**: 设定严格止损，避免极端情绪持续恶化导致的短期回撤
3. **综合验证**: 该信号需与OI Wall、Max Pain交叉验证
   - 若极端看跌情绪与关键支撑位（OI Wall）、最大痛点重合
   - 底部信号的有效性会进一步提升

### 图表展示规则

#### 7.8 图表类型
- **类型**: 面积图 + 折线图
- **PCR (OI)**: 蓝色面积 (#1890ff)，主曲线
- **PCR (Volume)**: 绿色虚线 (#52c41a)，辅助参考

#### 7.9 参考线
- **极度悲观线**: 1.5 (绿色虚线)
- **中性区间上限**: 1.2 (黄色点线)
- **中性区间下限**: 1.0 (黄色点线)
- **极度乐观线**: 0.8 (红色虚线)
- **现价线**: 蓝色垂直虚线

#### 7.10 数据展示
- **X轴**: 执行价 (Strike)，单位：美元
- **Y轴**: PCR值
- **总览面板**: 显示PCR值、情绪状态标签、OI分布进度条
- **区间参考**: 显示PCR区间划分参考

### 注意事项

#### 7.11 指标局限性
- PCR需结合VIX（波动率）、Gamma Exposure等多指标综合判断
- 单一指标易受短期交易噪音影响

#### 7.12 标的差异
- **指数PCR**（如标普500）更具宏观参考性
- **个股PCR**需结合个股基本面判定

#### 7.13 时间维度
- 未标注PCR的统计周期（日内/周/月）
- 短期（0DTE）和长期信号的强度不同
- 需结合周期进一步验证

## 模块 8: ATM IV (隐含波动率) 图表

### 功能描述
展示各执行价的隐含波动率分布（波动率微笑），计算ATM IV、HV、VRP等核心指标，分析波动率定价状态，并提供基于VRP的交易策略。

### 组件位置
- 图表组件: `src/components/IVChart.tsx`
- 计算工具: `src/utils/ivCalculator.ts`

### 核心概念

**ATM IV (At-The-Money Implied Volatility)**: 平值期权的隐含波动率，代表市场对未来波动的预期。数值越高，市场预期未来波动越大。

**HV (Historical Volatility)**: 历史波动率，代表标的资产过去一段时间实际发生的波动水平，作为基准反映当前波动的"真实水位"。

**VRP (Volatility Risk Premium)**: 波动率风险溢价，公式为 VRP = ATM IV - HV。正VRP表示市场预期波动高于历史波动，负VRP表示市场预期波动低于历史波动。

### 核心指标定义与数值解读

| 指标项 | 数值示例 | 核心定义 | 状态判定 |
|--------|----------|----------|----------|
| ATM IV | 24.83% | 平值期权的隐含波动率，代表市场对未来波动的预期 | 显著高于历史波动，预期波动偏激进 |
| HV | 18.80% | 标的资产过去一段时间的历史波动率 | 作为基准，反映当前波动的"真实水位" |
| VRP | +6.03% | 波动率风险溢价，VRP = ATM IV - HV | 正溢价，幅度达6.03%，属于中高溢价区间 |

### 计算公式

#### 8.1 基础公式
```
VRP = ATM IV - HV
VRP% = (VRP / HV) × 100%
```

#### 8.2 波动率微笑
- ATM期权IV最低
- 远离ATM时IV递增（OTM Put IV > OTM Call IV，因下行保护需求）

### VRP 状态判定

| VRP% 区间 | 状态 | 标签颜色 | 市场含义 |
|-----------|------|----------|----------|
| < 5% | 低溢价 | 绿色(#52c41a) | 波动率定价合理，预期与实际接近 |
| 5% - 15% | 正常溢价 | 浅绿(#73d13d) | 市场预期略高于历史波动 |
| 15% - 30% | 高溢价 | 黄色(#faad14) | 波动率定价偏贵，存在均值回归动力 |
| > 30% | 极端溢价 | 红色(#ff4d4f) | 波动率定价极端偏贵，市场情绪恐慌 |

### 正VRP的核心解读

**+6.03%的正VRP意味着**：
市场预期的波动率（IV）显著高于过去实际发生的波动率（HV），反映出期权交易者愿意支付溢价买入波动率，核心原因包括：

1. 对未来（如财报、美联储决议、地缘事件）的波动风险存在强烈担忧
2. 叠加负Gamma环境，做市商的对冲行为放大了波动预期，推高IV定价
3. 结合高PCR，散户集中买入看跌期权对冲风险，进一步推高Put期权IV

### 波动率定价的"虚高"判定

**24.83%的ATM IV较18.80%的HV高出6.03个百分点**：
- 溢价幅度约32%（6.03%/18.80%）
- 属于波动率定价偏贵的状态

**短期视角**：市场对波动的担忧存在情绪超调，波动率有向历史均值回归的动力
**中期视角**：若触发实质性风险事件，IV可能进一步上行；若风险落地，IV将快速回落

### 指标组合分析

| 指标组合 | 市场特征 | 核心结论 |
|----------|----------|----------|
| 负Gamma + 正VRP | 波动被放大 + 波动预期溢价 | 短期波动将持续处于高位，易出现"暴涨暴跌"的极致行情 |
| 高PCR + 正VRP | 极度悲观 + 波动率溢价 | 市场情绪处于"恐慌性定价"，反向看多胜率提升但需承受高波动 |

### 实操交易策略

#### 8.3 波动率卖方策略（适合风险承受能力中等，博弈均值回归）

**核心逻辑**：赌VRP收敛（IV向HV回归），赚取波动率溢价

**具体操作**：
- 卖出平值/虚值跨式期权（Short Straddle）
- 卖出宽跨式期权（Short Strangle）

**风控要点**：
- 负Gamma环境下波动剧烈，严格控制仓位（单策略仓位≤10%）
- 设置止损（如标的突破关键OI Wall支撑/阻力位时平仓）

#### 8.4 方向性反向看多策略（适合风险承受能力高，博弈情绪修复）

**核心逻辑**：PCR极端悲观是核心反向信号，正VRP虽代表波动溢价，但情绪修复会带动价格上行

**具体操作**：
- 买入虚值看涨期权（Call），而非直接买入现货（规避负Gamma带来的短期暴跌风险）
- 选择到期日1-2周的虚值Call，利用IV回落降低期权时间价值损耗

#### 8.5 中性对冲策略（适合保守型投资者）

**核心逻辑**：规避波动放大的风险，同时不放弃情绪修复的收益

**具体操作**：
- 买入标的现货 + 卖出虚值看涨期权（备兑开仓）
- 赚取期权权利金，抵消部分波动带来的回撤

### 图表展示规则

#### 8.6 图表类型
- **类型**: 折线图（波动率微笑）
- **Call IV**: 红色线 (#ff4d4f)
- **Put IV**: 绿色线 (#52c41a)
- **ATM IV**: 蓝色粗线 (#1890ff) + 浅蓝色面积

#### 8.7 数据展示
- **X轴**: 执行价 (Strike)，单位：美元
- **Y轴**: IV (%)，单位：百分比
- **现价线**: 蓝色垂直虚线标识当前价格位置

#### 8.8 总览面板

- VRP状态标签和溢价幅度
- VRP计算公式展示

### 关键注意事项

#### 8.9 事件驱动影响
- 若近期存在高影响事件（如美联储利率决议、标的财报），正VRP可能持续，甚至进一步扩大
- 需提前调整策略

#### 8.10 Gamma到期效应
- 临近期权到期日（尤其是0DTE期权），ATM IV会非线性波动
- VRP的收敛节奏会被打乱

#### 8.11 流动性风险
- 负Gamma环境下，标的流动性可能萎缩
- 期权买卖价差扩大，需选择流动性高的合约（买卖价差≤0.5%）

## 模块 9: SKEW (25Δ Risk Reversal) 图表

### 功能描述
展示波动率偏度（SKEW）分布，计算25Δ Risk Reversal值，分析市场对尾部风险（尤其是下行风险）的定价，并提供基于SKEW的交易策略。

### 组件位置
- 图表组件: `src/components/SkewChart.tsx`
- 计算工具: `src/utils/skewCalculator.ts`

### 核心概念

**SKEW (25Δ Risk Reversal)**: 是衡量期权市场对尾部风险（尤其是下行风险）定价的关键指标。核心逻辑是通过比较**25Δ看跌期权（Put）与25Δ看涨期权（Call）**的隐含波动率（IV）差异，反映市场对下行和上行尾部风险的定价倾斜。

**公式**：
```
SKEW (25Δ RR) = 25Δ Put IV - 25Δ Call IV
```

**核心逻辑**：
- 若 SKEW > 0：Put IV > Call IV，市场更担心下行风险，愿意为下行保护支付更高溢价
- 若 SKEW < 0：Call IV > Put IV，市场更担心上行风险

### 核心指标定义与数值解读

| 维度 | 数据/状态 | 核心解析 |
|------|-----------|----------|
| 核心数值 | +3.70% | 25Δ Put的隐含波动率比25Δ Call高3.70个百分点，Put IV显著溢价 |
| 偏斜程度 | 适度偏斜 | 3.70%的幅度属于"适度"区间，既反映了市场对下行风险的担忧，又未达到极端恐慌（极端SKEW通常>10%） |
| 保护属性 | 下行保护高于平均 | 市场对下行尾部风险的定价溢价，显著高于历史平均水平，说明投资者在主动买入Put对冲下跌风险 |

### SKEW 状态判定

| SKEW% 区间 | 状态 | 标签颜色 | 市场含义 |
|-----------|------|----------|----------|
| > 10% | 极端下行偏斜 | 红色(#ff4d4f) | 市场对下行风险极度担忧，处于恐慌状态 |
| 3% - 10% | 适度下行偏斜 | 浅红(#ff7875) | 市场对下行风险有合理担忧，偏斜适度 |
| -3% - 3% | 偏度正常 | 绿色(#52c41a) | 市场对上行和下行风险定价相对均衡 |
| -10% - -3% | 适度上行偏斜 | 浅绿(#73d13d) | 市场对上行风险更担忧（较少见） |
| < -10% | 极端上行偏斜 | 深绿(#389e0d) | 市场对上行风险极度担忧，处于极度乐观状态 |

### 市场含义与前期指标联动

结合PCR（1.755，极度悲观）、ATM IV（24.83%，波动率溢价）和负Gamma（-$42.13B，波动放大），SKEW指标的含义可进一步深化：

#### 9.1 情绪验证
+3.70%的SKEW与1.755的高PCR形成共振，验证了市场对下行风险的担忧，但SKEW的"适度"属性说明这种担忧是理性的，而非极端黑天鹅式的恐慌。

#### 9.2 波动定价
在ATM IV（24.83%）显著高于HV（18.80%）的背景下，SKEW的正偏斜说明波动率溢价主要由下行Put期权驱动，市场对"暴跌"的定价高于"暴涨"。

#### 9.3 做市商行为
负Gamma环境下，做市商的对冲行为会放大波动，而SKEW的正偏斜意味着做市商在对冲Put空头时，会在价格下跌时加速卖出标的，进一步强化下行趋势。

### 实操交易策略

#### 9.4 波动率套利策略（适合波动率交易者）

**核心逻辑**：利用Put IV相对贵、Call IV相对便宜的偏斜，构建风险逆转套利，博弈偏斜收敛。

**操作**：
- 卖出25Δ Put，买入25Δ Call
- 收取Put IV溢价，同时锁定上行收益

**风控**：
- 负Gamma环境下，若下行风险被触发，Put IV可能进一步飙升
- 将单策略仓位控制在10%以内
- 设置止损（如标的跌破关键支撑位时平仓）

#### 9.5 方向性优化策略（适合反向看多投资者）

**核心逻辑**：适度下行偏斜 + 高PCR，说明市场对下行的定价已部分反映，可利用Put IV溢价降低看多成本。

**操作**：
- 卖出虚值Put（如25Δ Put），收取权利金，若标的未跌破行权价，可赚取溢价
- 或买入虚值Call（利用Call IV相对便宜），降低看多成本

**合约选择**：优先选择流动性高的合约（买卖价差≤0.5%），避免流动性风险

#### 9.6 风险对冲策略（适合保守型投资者）

**核心逻辑**：利用SKEW的正偏斜，优化下行对冲成本。

**操作**：
- 若需对冲组合下行风险，可选择买入25Δ Put以外的更虚值Put（如10Δ Put）
- 其IV溢价相对较低，可降低对冲成本

### 图表展示规则

#### 9.7 图表类型
- **类型**: 折线图
- **Call Skew**: 红色线 (#ff4d4f)，相对于ATM的Call IV偏度
- **Put Skew**: 绿色线 (#52c41a)，相对于ATM的Put IV偏度

#### 9.8 参考线

#### 9.9 数据展示

- **总览面板**: 显示25Δ SKEW、25Δ Put IV、25Δ Call IV
- **SKEW解释**: 显示SKEW计算公式和含义

### 关键注意事项

#### 9.10 事件驱动影响
- 若近期出现高影响事件（如美联储决议、财报），SKEW可能快速飙升
- 偏斜程度加剧，需及时调整对冲策略

#### 9.11 流动性风险
- 负Gamma环境下，期权买卖价差可能扩大
- 需优先选择流动性高的合约，避免滑点损失

#### 9.12 指标局限性
- SKEW仅反映期权市场的定价
- 若出现实质性下行事件（如宏观数据恶化），偏斜可能进一步扩大
- 需结合基本面综合判断

## 模块 10: 多指标共振交易策略指南

### 功能描述
基于六大核心指标（GEX、Max Pain、OI Wall、PCR、ATM IV/VRP、SKEW）综合分析市场环境，生成可落地的交易策略建议。系统根据多指标共振原理，自动判定市场环境、分析交易信号，并提供具体的策略执行方案。

### 组件位置
- 策略组件: `src/components/StrategyGuide.tsx`
- 计算工具: `src/utils/strategyCalculator.ts`

### 核心前提与适用边界

**适用标的**: 优先标普500(SPX/SPY)、纳斯达克100(NDX/QQQ)等宽基指数/ETF；其次是日均期权成交量>100万张、市值>5000亿美元的大盘蓝筹（AAPL、MSFT等），禁止用于流动性差的小盘股。

**时间周期**: 核心适配近月到期合约（到期前1-15个交易日），尤其周度期权（3DTE/5DTE），中长线策略仅作辅助参考，0DTE仅限日内窄区间套利。

**核心铁律**: 所有开仓必须满足 **≥3个核心指标共振**，禁止单一信号开仓；策略必须严格匹配市场环境，禁止逆环境操作。

### 指标量化阈值与核心交易含义

| 指标 | 核心量化阈值与交易含义 | 关键补充规则 |
|------|------------------------|--------------|
| **GEX** | 极高正GEX: >20亿美元，钉住模式；中性正GEX: 5-20亿，温和震荡；中性: -5-5亿，方向不明；负GEX: <-5亿，波动放大；极端负GEX: <-20亿，恐慌/逼空模式 | 以SPX为基准，GEX符号直接决定做市商对冲方向，是环境判定的第一优先级 |
| **Max Pain** | 仅对到期前5个交易日内有效；当前价与Max Pain偏离度<3%(宽基)/<5%(大盘股)；曲线峰值对应强支撑/阻力 | 与OI Wall重合时支撑/阻力强度翻倍；正GEX环境下价格向Max Pain收敛概率>80% |
| **OI Wall** | 有效标准: 单一行权价OI是相邻平均OI的3倍以上；有效位置: Call Wall在当前价上方1%-3%，Put Wall在当前价下方1%-3% | 仅看近月到期合约OI；价格一次性突破有效OI Wall的概率<20% |
| **PCR** | 极度看空: >1.2；中性偏空: 0.9-1.2；中性: 0.7-0.9；中性偏多: 0.5-0.7；极度看多: <0.5 | 背离信号优先级更高: 价格创新高但PCR同步走高是强见顶信号 |
| **ATM IV/VRP** | ATM IV分位: <20%=极低波动，>80%=极高波动；高正VRP: >5个百分点，期权严重高估；负VRP: <-2个百分点，期权被低估 | 永远不要在高正VRP下裸买期权，不要在负VRP下裸卖期权 |
| **SKEW** | 正SKEW(RR>0): 看涨期权IV高于看跌，情绪极度乐观；中性: -1-+1；负SKEW(RR<-1): 看跌期权IV高于看涨，情绪恐慌 | 极端负SKEW+高PCR+负GEX是强触底信号；极端正SKEW+极低PCR+正GEX是强见顶信号 |

### 市场环境判定（先定模式，再选策略）

| 市场环境 | 核心判定标准 | 波动特征 | 策略大方向 |
|----------|--------------|----------|------------|
| **低波动震荡市** | 正GEX(>5亿) + ATM IV<40%分位 | 波动被压制，窄幅震荡，价格向Max Pain收敛，难突破OI Wall | 优先区间卖方策略，禁止单边裸买期权 |
| **高波动趋势市** | 负GEX(<-5亿) + ATM IV 40%-80%分位 | 波动放大，趋势性强，突破OI Wall会加速，Max Pain引力弱 | 优先方向性买方策略，禁止裸卖无保护期权 |
| **极端低波动市** | 极高正GEX(>20亿) + ATM IV<20%分位 | 波动极度收敛，日内振幅<0.5%，价格钉住Max Pain | 仅做窄区间蝶式套利，禁止任何方向性策略 |
| **极端高波动市** | 极端负GEX(<-20亿) + ATM IV>80%分位 | 暴涨暴跌，日内振幅>2%，支撑阻力极易失效 | 仅做对冲策略，禁止单边投机，优先空仓观望 |

### 多指标共振开仓信号（必须满足≥3个核心条件）

#### 10.1 看涨多头共振信号

**趋势条件**: 负GEX环境下，价格回踩有效Put Wall，且Put Wall与Max Pain重合；或正GEX环境下，当前价低于Max Pain，偏离度<2%，有明确向上收敛空间。

**情绪条件**: OI PCR>0.9（中性偏空及以上），最佳为>1.2的极度恐慌值，出现价格新低但PCR不创新高的底背离。

**波动率条件**: VRP<0（期权被低估），或ATM IV处于20%-60%分位，无高溢价；SKEW(RR)<-1，市场恐慌定价充分。

**空间条件**: 上方最近的Call Wall距离当前价>2%，上行无强阻力，上涨空间充足。

#### 10.2 看跌空头共振信号

**趋势条件**: 负GEX环境下，价格触及有效Call Wall，且Call Wall与Max Pain重合；或正GEX环境下，当前价高于Max Pain，偏离度<2%，有明确向下收敛空间。

**情绪条件**: OI PCR<0.7（中性偏多及以下），最佳为<0.5的极度贪婪值，出现价格新高但PCR不创新低的顶背离。

**波动率条件**: VRP<0（期权被低估），或ATM IV处于20%-60%分位，无高溢价；SKEW(RR)>1，市场乐观定价过度。

**空间条件**: 下方最近的Put Wall距离当前价>2%，下行无强支撑，下跌空间充足。

#### 10.3 区间震荡共振信号

**波动条件**: 正GEX环境(>5亿)，GEX数值越高，区间约束越强。

**边界条件**: 上方有有效Call Wall，下方有有效Put Wall，形成1%-3%的明确价格运行区间。

**收敛条件**: Max Pain处于Call Wall和Put Wall的区间内部，价格有向中间收敛的动力。

**波动率条件**: VRP>2（期权有溢价），ATM IV处于40%-80%分位，卖方有时间价值优势；SKEW处于中性区间，无极端尾部风险预期。

### 具体期权操作策略

#### 场景1: 低波动震荡市 + 区间震荡共振信号

**核心策略**: 铁鹰套利/卖出宽跨式，赚时间价值与价格收敛的钱

**合约选择**: 到期前3-7个交易日的周度合约，越临近到期，时间价值衰减越快，Max Pain引力越强。

**行权价设置**: 
- 基础卖出端: 卖出上方有效Call Wall价位的看涨期权、下方有效Put Wall价位的看跌期权
- 无明确OI Wall时: 以Max Pain上下1%为卖出行权价
- 风险保护端: 在卖出看涨上方0.5%、卖出看跌下方0.5%，分别买入对应行权价的期权，构建铁鹰套利

**仓位控制**: 单策略保证金占用不超过总账户资金的5%，极高正GEX环境下不超过10%，禁止无保护裸卖期权。

**入场时机**: 价格处于区间中间位置，或接近区间上沿/下沿，偏离Max Pain不超过2%。

**止盈止损**: 
- 止盈: 期权剩余价值不足开仓时的20%，或到期前1天平仓，赚80%利润即离场
- 止损: 价格突破Call Wall/Put Wall超过0.5%且30分钟不收回，立即平仓，最大亏损不超过初始权利金的2倍

#### 场景2: 高波动趋势市 + 看涨/看跌共振信号

**核心策略**: 垂直价差/裸买期权，赚价格趋势与波动率上升的钱

**通用基础规则**: 
- 合约选择: 到期前7-15个交易日的合约，规避0DTE的极端时间损耗
- 仓位控制: 单策略权利金支出不超过总账户资金的2%，禁止重仓裸买期权

**看涨多头策略执行细节**:

**行权价设置**:
- 保守型: 选平值(ATM)看涨期权(delta=0.5)，兼顾delta方向收益与vega波动率收益
- 进取型: 选25delta虚值看涨期权，性价比最高，适配强趋势行情
- 稳健型: 构建牛市看涨垂直价差，买入ATM看涨期权，同时卖出上方临近Call Wall价位的看涨期权

**入场时机**: 价格回踩Put Wall获得支撑，出现15/30分钟级别企稳信号，PCR处于高位，VRP为负。

**止盈止损**:
- 止盈: 价格达到目标位(Max Pain/下一个OI Wall)，或单笔盈利达到100%，分批止盈50%，剩余仓位设置移动止损
- 止损: 价格跌破Put Wall超过0.5%且30分钟不收回，或单笔亏损达到50%，立即止损，绝对不扛单

**看跌空头策略执行细节**:

**行权价设置**:
- 保守型: 选平值(ATM)看跌期权(delta=-0.5)
- 进取型: 选25delta虚值看跌期权
- 稳健型: 构建熊市看跌垂直价差，买入ATM看跌期权，同时卖出下方临近Put Wall价位的看跌期权

**入场时机**: 价格触及Call Wall遇阻，出现15/30分钟级别回落信号，PCR处于低位，VRP为负。

**止盈止损**: 与看涨策略规则一致。

#### 场景3: 极端低波动市

**核心策略**: 窄区间蝶式套利，仅赚到期价格收敛的钱

**合约选择**: 到期前1-3个交易日的合约，最佳为1DTE/0DTE，极致博弈时间衰减。

**行权价设置**: 以Max Pain为中间行权价，上下各0.5%的行权价构建对称蝶式套利，买入上下两侧虚值期权，卖出中间平值期权。

**仓位控制**: 权利金支出不超过总账户资金的1%，盈利与亏损均有限。

**入场时机**: 开盘后1小时，价格波动幅度<0.3%，确认进入钉住模式。

**止盈止损**:
- 止盈: 到期前1小时，价格在区间内则平仓止盈
- 止损: 价格突破区间上下沿超过0.3%，立即止损，最大亏损为初始投入的全部权利金

#### 场景4: 极端高波动市

**核心策略**: 仅做对冲，禁止单边投机，优先空仓观望

**仅有的合规可操作策略**:

**保护性对冲**: 持有股票多头时，买入5delta虚值看跌期权做尾部保护，到期日选择1-3个月，规避短期高IV的损耗。

**波动率跨式套利**: 仅当VRP为负、IV处于历史高位但预期继续飙升时，买入平值跨式期权（同时买ATM看涨+看跌），赚波动率上升的钱，权利金支出不超过总账户资金的1%。
